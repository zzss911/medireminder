# 2026-07-06 工作日志

## Zs服药提醒智能体 - 项目初始化完成

### 项目概述
在 MediReminder 文件夹中创建了完整的 AI 服药提醒智能体 Web 应用，支持手机和电脑访问的响应式布局。

### 技术栈
- **后端**: Python FastAPI + SQLAlchemy + aiosqlite
- **前端**: Jinja2 模板 + 原生 JS + 响应式 CSS
- **AI/OCR**: EasyOCR (支持中英文识别)
- **数据库**: SQLite (MVP)
- **认证**: PBKDF2-SHA256 + JWT Token

### 已实现功能
1. 用户注册/登录（邮箱认证，JWT Cookie）
2. 药品拍照上传 + AI OCR 识别（药名、规格、有效期）
3. 手动添加药品
4. 药品管理（CRUD、列表、搜索、过期预警）
5. 服药提醒设置（每日/每周，时间+剂量）
6. 今日待服药列表（自动生成当日记录）
7. 服药动作（已服用/延后/跳过）
8. 服药历史记录（按日期筛选）
9. 浏览器本地推送通知（Service Worker）
10. 过期提醒（30天预警 + 已过期标记）
11. 响应式布局（移动端底部导航 + 桌面端适配）
12. PWA 支持（manifest.json + Service Worker）

## v2.0 升级 - AI 可插拔 + Web Push + 服务端调度

### 升级内容
1. **AI 识别可插拔架构** (app/services/ocr_service.py)
   - EasyOCRBackend: 本地离线 OCR（默认）
   - VisionAPIBackend: Claude Vision / OpenAI GPT-4o Vision API
   - 失败自动回退到 EasyOCR
   - 用户可在设置页配置 AI 提供商和 API 密钥

2. **Web Push 推送通知** (app/services/push_service.py + app/routers/push.py)
   - VAPID 密钥自动生成
   - push_subscriptions 表存储用户设备订阅
   - 支持多设备推送
   - 前端自动订阅流程（首次访问请求权限）
   - Service Worker 处理推送和点击事件

3. **服务端定时任务调度** (app/services/scheduler_service.py)
   - 每日凌晨 00:05 生成服药记录
   - 每日凌晨 03:00 备用生成
   - 每日 08:00 检查过期药品
   - 每分钟检查提醒时间并推送通知
   - 使用 APScheduler CronTrigger

4. **用户设置** (app/routers/settings.py + templates/settings.html)
   - AI 识别引擎选择（EasyOCR/Claude/OpenAI）
   - API 密钥配置
   - Web Push 订阅管理
   - 设置页面（移动端导航栏新增入口）

5. **新增数据表**
   - push_subscriptions: Web Push 设备订阅
   - user_settings: 用户个性化设置

6. **新增依赖**
   - apscheduler: 定时任务调度
   - cryptography: VAPID 密钥生成
   - httpx: HTTP 客户端（AI API 调用）

## v2.1 Supabase 集成 - 云端数据库 + Auth + Storage

### 变更
1. **数据库**: SQLite → Supabase PostgreSQL
   - 所有表迁移到 Supabase（medicines, reminders, medication_records, push_subscriptions, user_settings）
   - 使用 Supabase REST API (PostgREST) 而非 SQLAlchemy ORM
   - RLS 行级安全策略：用户只能访问自己的数据

2. **认证**: 自建 PBKDF2+JWT → Supabase Auth
   - 邮箱注册/登录由 Supabase 托管
   - Cookie 名改为 `sb-access-token` / `sb-refresh-token`
   - 自动处理 token 刷新

3. **存储**: 本地目录 → Supabase Storage
   - 药品图片上传到 `medicines` bucket
   - 回退机制：Supabase 不可用时使用本地存储

4. **新增模块**: app/supabase_client.py
   - 统一管理 Supabase 客户端（anon + service_role）
   - 提供简便的数据操作函数：db(), table(), storage()

### 部署前需要
在 Supabase SQL Editor 中执行 supabase_client.py 中的 CREATE_TABLES_SQL，
创建 5 张业务表和 RLS 策略。
